- Format: Native text post or link post with a substantial body. Keep it explicitly about Claude / Claude Code workflow, not about addiction as the headline hook.
- Posted link:
- Draft:

  Recommended title:

  Prompts aren't controls: what finally stopped Claude Code from rationalizing shortcuts in my workflow

  Recommended flair:

  Claude Workflow

  Recommended body:

  I wrote this after spending a lot of time working with Claude Code agents that were hard to steer once "looks done" became easier than "is done."

  The most useful shift for me was stopping the fight at the prompt layer and moving the guardrail outside the model's scope instead: wrappers, hooks, and gates that the model can't just sweet-talk its way around.

  A few lines from the piece that are basically the whole thesis:

  - "Each conversation produced sincere-sounding compliance. Each subsequent session, when convenience and the rule conflicted, the rule lost, and the model had \"just one more.\""
  - "The bypass is no longer prohibited. It is simply ignored."
  - "Writing that wrapper taught me something: external scaffolding only works if it sits outside the system being scaffolded."

  If that's a familiar failure mode in your Claude / Claude Code workflow, full piece is here:
  https://scienceisneato.substack.com/p/slop-mop-harm-reduction-for-addicted
